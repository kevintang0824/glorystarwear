## Project Type

- Pure static website: HTML, CSS, JavaScript, SVG, JPG images.
- No build step required.
- Can be deployed directly to GitHub Pages, Netlify, Vercel, Cloudflare Pages, or any standard web host.

4. Select the branch and root folder.
5. After publishing, test homepage, product pages, sitemap, and contact links.

- Build command: leave empty.
- Publish directory: project root, usually `.`.
- Framework preset: static / no framework.

### Analytics Hook

`script.js` initializes `window.dataLayer` without loading an analytics vendor. A future Google Tag Manager or analytics integration can consume these events:

- `session_landing`
- `contact_click`
- `quote_start`
- `quote_submit_email`
- `quote_submit_whatsapp`
- `quote_copy_brief`
- `resource_download`
- `catalog_filter`
- `catalog_search`

Campaign attribution keeps the initial landing URL, referrer, `utm_*`, `gclid`, and `msclkid` values in session storage. It also adds the attribution context to the prepared email or WhatsApp inquiry. Do not send names, email addresses, phone numbers, or free-text project details into analytics.

### IndexNow

Deploy the public key file before submitting URLs:

```text
https://glorystarwears.com/8022fa20d2ef4befc52093d274ae7687.txt
```

After changed pages are live, submit only their canonical URLs:

```bash
node scripts/submit-indexnow.mjs \
  https://glorystarwears.com/resources/ \
  https://glorystarwears.com/resources/oem-vs-odm-sportswear.html \
  https://glorystarwears.com/resources/private-label-activewear-moq.html \
  https://glorystarwears.com/resources/custom-sportswear-tech-pack.html \
  https://glorystarwears.com/resources/sportswear-sample-approval-checklist.html \
  https://glorystarwears.com/resources/activewear-size-grading-guide.html \
  https://glorystarwears.com/resources/teamwear-roster-packing-guide.html
```

IndexNow notifies participating search engines that a URL changed. It does not guarantee crawling, indexing, or ranking. Do not submit the live URLs until the matching files have been deployed.

`llms.txt` is a concise reference for systems that choose to read it. It is not a replacement for crawlable HTML, `robots.txt`, the sitemap, or Search Console, and Google Search does not use it as an AI visibility signal.

Before going live, verify that `https://glorystarwears.com/` is the final domain. If not, replace it everywhere.

## Maintenance Notes

- Add new product images to `assets/images/`.
- Use descriptive lowercase filenames with hyphens.
- Keep an AVIF source alongside every JPEG; run `python3 scripts/generate_hero_avif.py --all-images` after adding or replacing website images.
- Keep image references relative in HTML, such as `../assets/images/example.jpg`.
- Keep all important product images listed in `sitemap.xml`.
- Use accurate sitemap `lastmod` dates only when a page changes meaningfully.
- When adding a new page, add it to navigation, mobile navigation, footer links if needed, `sitemap.xml`, and related JSON-LD.
- After deploying a changed canonical page, submit that URL through IndexNow and inspect important pages in Search Console.
